# Sprint Scope

## In Scope

- CSV/sample data ingestion
- Basic production data model
- Risk detection rules
- Schedule-impact reasoning
- Daily brief generation
- Owner-specific questions
- Simple API or local dashboard
- Markdown and mock Slack-style output
- Realistic demo data

## Out of Scope

- Real ERP, MES, QMS, or Slack integrations
- Authentication and permissions
- Multi-tenant database
- Complex dashboards
- Autonomous task assignment
- Long-term memory
- Production deployment hardening

## Product Test

A manufacturing leader should be able to look at the generated brief and quickly answer:

**What needs my attention today?**
